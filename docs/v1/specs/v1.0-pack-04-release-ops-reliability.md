# Pack 04: Release/Operations Reliability (G4, G8, G9, G11)

**Status:** In Progress  
**Start date:** 2026-04-15  
**Related gates:** `G4`, `G8`, `G9`, `G11`  
**Blocking:** Pack 04 закрывает `G4`, `G8`, `G9` (`blocking_v1 = true`) и закрывает `G11` ( `blocking_v1 = false`).

## Purpose

Pack 04 делает эксплуатационный контракт выпуска и жизни в проде:

- поддерживаемые платформы явно задокументированы и проверены;
- установочный пайплайн верифицируем по `.sha256` до записи в целевой `BINDIR`;
- install/upgrade/uninstall сценарии не имеют невидимых регрессий и имеют диагностируемые пути отказа;
- в документированном runbook есть recovery-гайды по самым частым operational-failure сценариям.

## Scope

### Included

1. **G4:** Platform support matrix completion and test-backed evidence.
   - Linux + macOS как поддерживаемые;
   - WSL2 usage как expected в Linux-подобном профиле;
   - Native Windows как explicit unsupported.
   - Проверка привязана к `docs/platform-support.md` + тестам.
2. **G8:** Release artifact integrity during install.
   - Проверить `install.sh` на download `.sha256`;
   - fail-closed при mismatch/missing checksum;
   - fallback strategy между `sha256sum` и `shasum`.
3. **G9:** Install/upgrade/uninstall operational proof.
   - Проверить базовую install-настройку wrapper;
   - проверку повторного install как upgrade-safe/idempotent path;
   - проверку uninstall очистки.
4. **G11:** Troubleshooting/recovery baseline.
   - add/update doc with common failure + recovery playbook (no feature changes).

### Excluded

- Any functional security/parser work (Track A/B scope).
- New installers for additional OS families.
- Feature expansion of snapshot providers.

## Acceptance Criteria

### AC1 (G4): support matrix and policy are explicit + test-backed

1. `docs/platform-support.md` явно перечисляет:
   - Linux / macOS / Windows-host via WSL2 / Windows (unsupported);
   - что `bash`/`zsh` + POSIX model поддерживаются;
   - почему Windows unsupported and what is deferred.
2. `README.md` содержит ссылку на политику поддержки и ясное разделение WSL2/native Windows.
3. `rtk cargo test --test platform_support_docs` зелёный, включая проверки текста платформенной политики.

### AC2 (G8): checksum verification is mandatory for install artifact

1. `scripts/install.sh` загружает бинарник и соответствующий `.sha256` asset.
2. Проверка целостности выполняется до установки в `BINDIR`.
3. Невыполнение проверки не трогает `BINDIR`/установку.
4. Поддержка fallback верификации:
   - `sha256sum` если доступен;
   - `shasum` как fallback.
5. Документирование команды/процедуры в `README.md` отражает обязательность верификации.

### AC3 (G9): install/upgrade/uninstall flow is documented and behaviorally validated

1. `README.md` сохраняет install/uninstall инструкции и явно показывает recovery options.
2. `tests/installer_flow.rs` содержит проверки:
   - `install_script_configures_shell_wrapper_block_once` (базовый install + идемпотентный повторный запуск),
   - `install_script_prefers_aegis_real_shell_when_shell_already_points_to_wrapper` (recursion-safe),
   - `install_script_rejects_checksum_mismatch_before_touching_bindir`,
   - `install_script_rejects_missing_checksum_before_touching_bindir`,
   - `install_script_fails_when_no_supported_checksum_tool_exists`,
   - `install_script_falls_back_to_shasum_when_sha256sum_is_missing`,
   - `uninstall_script_removes_managed_block_and_binary`.
3. В pack-логике “upgrade” трактуется как повторный запуск `install.sh` после существующего wrapper/конфигурации.

### AC4 (G11): recovery guidance exists for common operational failures

1. `docs/troubleshooting.md` добавлен и содержит минимум:
   - install failed: checksum mismatch / missing checksum;
   - recursive-shell/setup failures (AEGIS_REAL_SHELL/`SHELL` mismatch);
   - missing permissions during install/uninstall (`/usr/local/bin`, rc-файлы);
   - rollback/restore actions for installer misconfiguration.
2. Документ содержит короткие команды проверки/восстановления:
   - как проверить/перезапустить shell wrapper;
   - как удалить wrapper и binary;
   - где смотреть stderr для `aegis rollback` и как действовать дальше.
3. Link from `README.md` and docs index so users can find it.

## Rollback / stop conditions

- Если fail-closed инварианты `G8`/`G9` не выполняются (silent success при bad checksum, partial install success), pack переводится в `blocked` до исправления.
- Если `README.md`/`docs/troubleshooting.md` теряет актуальность после изменения installer/policy, gate нельзя закрывать без их синхронизации.
